# Non-Claims

Ghost Ark deliberately does not claim:

- That a signed receipt proves the factual truth of an underlying claim.
- That a curated dataset is statistically valid merely because it is reproducible.
- That a tenant boundary eliminates all governance risk.
- That AWS service-level immutability equals legal immutability.
- That search results are complete without catalog and lineage verification.
- That a compliance export is certification by itself.
- That Ghost Ark provides legal, clinical, therapeutic, emotional-safety, or mental-health assurance.
- That a PASS verdict means safe.
- That runtime attestation proves production Nitro Enclave isolation before live enclave attestation is integrated and validated.
- That the receipt proof interface provides production zero-knowledge privacy before a real proof backend is integrated and verified.
- That the counterexample engine is complete AWS IAM formal verification.

Ghost Ark claims only what its evidence path can support: observed object metadata, deterministic canonical payloads, KMS-backed signatures, ledger state, policy configuration, and replayable workflow records.
