# Claim/Evidence Matrix

This matrix maps each public Ghost-Ark claim to its allowed wording, forbidden wording, evidence location, and validation status. A claim may only be made in the allowed form, and only while the listed evidence exists and is reproducible.

Status vocabulary:

- `Local evidence` — reproducible on a laptop with zero AWS credentials.
- `Local partial` — local artifacts exist; the claim boundary is narrower than the component name suggests.
- `AWS-synth-only` — infrastructure templates synthesize locally; no live runtime evidence.
- `AWS-required` — the claim cannot be validated without a bounded live AWS window.
- `Not implemented` — no artifact exists; the row is a placeholder boundary.

## Interpretation rules

- A row marked `AWS-required` must not be presented as validated from local tests, fixtures, or CDK synthesis.
- A row marked `Not implemented` is a boundary placeholder; it must not appear in public claims at all.
- Forbidden wording in this table is quoted only to define what reviewers should reject.
