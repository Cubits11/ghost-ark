# Risk Register

This register tracks governance and evidence risks for Ghost-Ark. Severity reflects the damage if the risk materializes, not implementation difficulty. High risks stay high until concrete evidence closes the gap.

Severity vocabulary: High, Medium, Low. Status vocabulary: Open, Mitigating, Closed.

| Risk ID | Risk | Severity | Evidence gap | Mitigation | Status | Spine |
|---|---|---|---|---|---|---|
| RISK-001 | Overclaiming certification or AI safety in public surfaces | High | No automated gate existed on all public claim surfaces until the claim scanner; scanner coverage must not regress | Claim scanner (`npm run scan:claims`) with exact-path allowlist; claim/evidence matrix; README claim-discipline section | Mitigating | Spine A |
| RISK-002 | Local evidence mistaken for AWS proof | High | Local tests and fixtures exercise mock/HMAC-dev paths; no live evidence bundles preserved | Explicit local-only / AWS-synth-only / AWS-required labels; synthetic fixture is schema-locked to `NOT_RUN`; validator rejects live relabeling | Mitigating | Spine A / C |
| RISK-003 | Tenant isolation requires live API/Cognito/IAM validation | High | No preserved live evidence that deployed authorizers and IAM policies enforce tenant boundaries | Keep CLAIM-006 marked AWS-required; plan bounded live isolation test in a future window | Open | Spine C |
| RISK-004 | KMS provenance requires live AWS validation | High | Local signer tests use HMAC-dev; no preserved live KMS sign/verify evidence | Keep CLAIM-007 marked AWS-required; use immutable key ARNs in verification-critical paths | Open | Spine C |
| RISK-005 | Receipt schema drift breaking verifier compatibility | Medium | No formal schema-migration procedure for semantic field changes (e.g. `action_taken` multiplicity) | Receipt v1 compatibility rule in CLAUDE.md; schema tests; require migration before semantic changes | Mitigating | Spine B |
