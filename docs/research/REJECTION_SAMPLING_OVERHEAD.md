# The Computational Overhead of Constrained Decoding

## Abstract
We study the computational overhead of constrained decoding in autoregressive transformer inference under a fixed hardware and decoding protocol. Specifically, we analyze exact rejection sampling, where a proposal distribution $P$ is filtered to produce samples from a target distribution $Q$ subject to an envelope bound $Q(x) \le M P(x)$, with $M = \exp(D_\infty(Q\|P))$. Under this protocol, the expected number of proposal samples per accepted token is $M$, so greater proposal-target mismatch increases the expected number of forward passes and may increase measured energy per committed token. We further hypothesize that, on real accelerator hardware, rejected proposals induce measurable overhead through repeated kernel execution, cache maintenance, and state rollback, producing an energy cost that can be empirically estimated from power telemetry and memory-traffic traces.

## Main Claim
Let $P$ denote the native proposal distribution produced by the model and $Q$ the constrained target distribution enforced by an external gate. If sampling is implemented by exact rejection sampling, then the acceptance rate is $1/M$, where $M$ is the smallest envelope constant satisfying $Q(x) \le M P(x)$ for all $x$. Therefore, the expected number of forward passes per accepted token is $M$, and any increase in $D_\infty(Q\|P)$ increases expected inference work through the rejection mechanism. This is an algorithmic overhead result, not a universal thermodynamic law about semantics or safety.

## Defensible Hypothesis
Under fixed batch size, fixed temperature, fixed model weights, and exact rejection-sampling decoding, the measured energy per accepted token increases monotonically with the mismatch between $P$ and $Q$, as quantified by $D_\infty(Q\|P)$ or an equivalent envelope constant. The mechanism is straightforward: more rejected proposals require more forward passes, more memory traffic, and more control-flow overhead on the accelerator. Any Landauer-style discussion should be limited to logically irreversible state disposal in the implementation and should not be presented as a direct per-rejection law unless the specific hardware erasure path is identified.

## Experimental Protocol
Use batch size 1, a fixed model, and a fixed exact rejection-sampling decoder. Instrument core power, HBM traffic, and kernel timestamps, and compare accepted-token energy across a controlled family of target distributions $Q$ that progressively diverge from the proposal $P$. Report acceptance rate, mean proposals per accepted token, joules per accepted token, and memory-traffic deltas; these are the observable quantities supported by the protocol. Avoid claims about “hallucination,” “agency,” or “semantic failure,” because those are interpretive layers, not physical observables.

## Suggested Theorem Statement
If an autoregressive inference system samples by exact rejection sampling from proposal $P$ under target $Q$, and if $Q(x) \le M P(x)$ for all $x$, then the expected number of proposal evaluations per accepted sample is $M$, and any hardware implementation whose per-proposal energy is bounded below by $E_{\mathrm{fwd}}$ has expected energy per accepted sample at least $M E_{\mathrm{fwd}}$. This lower bound is algorithmic and implementation-dependent, not a general thermodynamic bound on meaning or correctness. If the implementation performs irreversible rollback of speculative state, that rollback may add extra dissipative cost, but this should be measured directly rather than assumed from first principles.

## Reviewer Notes
The safest framing is to present this as a study of energy overhead induced by constrained sampling rather than a theory of safety or cognition. The strongest version you can defend is: stricter exact constraints raise rejection rates, which raise forward-pass count, which can raise energy use on real accelerators. Anything stronger should be treated as a conjecture until backed by hardware-level measurements.
