The weakness we measured

§5.5 · live

Most papers state a caveat and stop. The ledger's tombstone set is capacity-bounded, so pruning reopens a replay window — and we measured it. Driving the real ledger yields an exact law: window = max(0, K − C). Below capacity the window is empty; above it, every extra tombstone is one more replayable nonce.

The boundary is the point

What Ghost-Ark does not claim

non-claims

A skeptical reviewer should be able to distrust everything and still verify the record. These are boundaries we consider part of the contribution — not an escape hatch.

  • —No semantic safety. The gate bounds a function of supplied marginals; garbage in, well-computed garbage out. It certifies no detector's hit rate.
  • —Bounded models, not verified implementations. TLC is exhaustive only within stated bounds; conformance of the Rust/TypeScript code is tested, not proved.
  • —DEV key, not production custody. A local ed25519 key — not KMS, HSM, TPM, or Nitro attestation. Signing proves authorization over bytes, not runtime integrity.
  • —Modeled attacker only. The zero-advantage result covers the nine-attack / four-game family, in-process — not unmodeled attacks or availability.
  • —No live-cloud evidence. KMS-mode signing and the full AWS path are design targets; nothing here is a cloud measurement.
  • —Single node. Ledgers are single-writer; multi-node consensus is future work, not a property of this system.
